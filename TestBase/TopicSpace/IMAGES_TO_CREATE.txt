================================================================================
CREATION STRATEGY
================================================================================

PHASE 1 (Week 1): Create Priority 1 images (Diagrams 1-6)
  - These are used most frequently
  - Most critical for understanding
  - Estimated time: 3-4 hours

PHASE 2 (Week 2): Create Priority 2 images (Diagrams 7-12)
  - High value for learning
  - Support key concepts
  - Estimated time: 4-5 hours

PHASE 3 (Week 3+): Create Priority 3 images (Diagrams 13-20)
  - Reference and supporting material
  - Nice to have but not critical
  - Estimated time: 3-4 hours

RECOMMENDED TOOLS:
- Circuit diagrams: LTspice, Circuit Lab, or Draw.io
- Graphs/curves: Desmos, Matplotlib, or Excel
- Vector graphics: Inkscape (free) or Adobe Illustrator
- Photos/annotations: Snagit or basic image editor
- Hand diagrams: Draw.io, Procreate, or hand-drawn + scanned

FILE FORMAT RECOMMENDATIONS:
- PNG for diagrams (preserves quality, no loss)
- High resolution (300 DPI minimum for print quality)
- Consistent sizing (all graphs same size if possible)
- Consistent color scheme (use palette for brand consistency)

PLACEMENT IN DOCUMENT:
- Insert image placeholder text "[ADD IMAGE: description]" in text file
- Link to image files in final document
- Number images for cross-referencing
- Include figure captions

================================================================================
SPECIFIC TECHNICAL NOTES FOR EACH DIAGRAM
================================================================================

Power Triangle:
- Keep simple, avoid 3D effects
- Color: Red=P, Blue=I²R, Green=V²/R
- Include both valid and invalid (P=V²R crossed out)

Series/Parallel Circuits:
- Use standard IEEE symbols
- Color: Red=+, Black=Ground, White=other
- Include ground symbols for clarity
- Label all voltages and currents

RC Curves:
- Exponential curves, NOT linear
- Mark key percentages (63%, 95%, 99%)
- Time axis in τ units, not seconds
- Include mathematical formula on graph

Component Symbols:
- Use standard symbols per IEEE 315
- Organize logically (passive first, then active)
- Include component name below each symbol
- Size all symbols consistently

LED Color Chart:
- Use actual LED colors or close approximations
- Show voltage ranges as bars
- Include both typical and max values
- Color intensity could represent brightness

Right-Hand Rule:
- Show actual hand in realistic positions
- Multiple views for each context (top, side, perspective)
- Use arrows to show relationships between vectors
- Label each finger/thumb with meaning

Circuit Analysis Flowchart:
- Use diamond shapes for decisions
- Use rectangles for processes
- Include "yes/no" labels on decision branches
- Show end state clearly

Multimeter Connections:
- Show three separate circuits (voltage, current, resistance)
- Highlight meter connections in red
- Show correct dial setting for each
- Include warning symbols for improper usage

Breadboard Layout:
- Show overhead view
- Color power rails red/black
- Show component placement with labels
- Highlight proper wire routing (short, organized)
